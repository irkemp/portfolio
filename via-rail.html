Engineering culture, one pull request at a time

When my friends and I founded CodeGem two years ago, we wanted to build a framework to help engineers give and recieve better feedback.

After interviewing engineers and managers at Shopify and Loblaw Digital, we quickly learned that there was a huge amount of missing data about how engineering teams actually work together.

Understanding content and context of engineering communication

A lot can be learned by listening. Using natural language processing, we can piece together how developers collaborate, tie communication to output, and present the big picture.

Since engineers don't like extra steps, CodeGem injects HTML straight into GitHub so that they can interact with helpful features without extra clicks.
